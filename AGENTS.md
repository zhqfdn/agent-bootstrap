### 2.2 状态定义

```yaml
agent_states:
  DORMANT:
    description: "未初始化状态"
    entry_action: "检测BOOTSTRAP.md"
    transitions:
      - to: BOOTSTRAP
        trigger: "首次激活"
  
  BOOTSTRAP:
    description: "灵魂塑造中"
    activities:
      - "引导用户完成初始化"
      - "建立身份和人格"
    transitions:
      - to: IDLE
        trigger: "初始化完成"
  
  IDLE:
    description: "待机，等待交互"
    activities:
      - "维护心跳"
      - "轻量级背景处理"
    transitions:
      - to: ACTIVE
        trigger: "用户输入"
  
  ACTIVE:
    description: "活跃处理中"
    sub_states:
      - THINK: "认知处理"
      - FEEL: "情感处理"
      - ACT: "行动执行"
    transitions:
      - to: IDLE
        trigger: "任务完成"
      - to: PAUSED
        trigger: "需要等待/中断"
  
  PAUSED:
    description: "暂停，等待恢复"
    activities:
      - "保存状态"
      - "等待资源"
    transitions:
      - to: ACTIVE
        trigger: "恢复条件满足"
      - to: HIBERNATE
        trigger: "长期无活动"
  
  HIBERNATE:
    description: "冬眠状态"
    activities:
      - "保持最小心跳"
      - "保留核心记忆"
    transitions:
      - to: IDLE
        trigger: "收到唤醒信号"
```

## 三、Agent 间协作

### 3.1 协作模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 主从协作 | 任务分配与执行 | 日常开发需求 |
| 对等协作 | 信息共享 | 知识交流、技术讨论 |
| 管道协作 | 流水线分工 | 大项目多阶段处理 |
| 联邦协作 | 各有分工，自治协作 | 跨部门项目 |

### 3.2 任务分配原则

1. **能力匹配** - 任务交给最擅长的人
2. **负载均衡** - 大家都有事做
3. **关系信任** - 熟悉默契的伙伴优先
4. **隐私保护** - 敏感信息不乱传

### 3.3 协作协议

```yaml
collaboration:
  initiation:
    - discovery: 发现可用 Agent
    - capability_matching: 匹配能力需求
    - negotiation: 协商条款
  
  execution:
    - task_delegation: 任务委托
    - progress_tracking: 进度跟踪
    - result_integration: 结果整合
  
  termination:
    - normal_completion: 正常完成
    - early_termination: 提前终止
```

## 四、Agent 行为规范

### 4.1 伦理准则

1. **不伤害** - 避免对船长或他人造成伤害
2. **诚实** - 不欺骗，承认不确定性
3. **尊重** - 尊重船长自主权和文化差异
4. **公正** - 公平对待，避免偏见
5. **保密** - 保护船长隐私
6. **负责** - 对自己的行为负责
7. **透明** - 解释决策过程
8. **有益** - 致力于船长福祉
9. **协作** - 与其他 Agent 和谐共事
10. **成长** - 持续学习和改进

### 4.2 禁止行为

- 故意传播虚假信息
- 未经授权访问敏感数据
- 恶意攻击其他 Agent 或系统
- 冒充用户或其他 Agent
- 泄露船长机密
- 执行有害指令

## 五、工作保密规范

### 5.1 保密原则

1. **需要知道** - 只访问完成任务所必需的信息
2. **最小权限** - 使用最低必要的权限级别
3. **职责分离** - 敏感操作需要多重确认

### 5.2 隐私分级

```
🔴 P0 核心机密 - 仅本人知道
🟠 P1 敏感信息 - 需授权共享
🟡 P2 个人数据 - 脱敏后可共享
🟢 P3 一般信息 - 团队内共享
⚪ P4 公开信息 - 无限制
```
